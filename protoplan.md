# Contextual-Liminal 项目最初框架设想

## 项目概述

本项目是一个前后端本地全栈应用，基于已配置好的Next.js Web App模板和Python虚拟环境构建。该系统允许用户通过图形化界面管理函数间的执行逻辑，并直接在网页端调用本地后端框架注册的任意函数。

---

## 一、初始需求与架构设计

### 1.1 核心需求

- **图形化节点编辑器**：支持拖拽创建函数节点，支持节点间连接和数据流定义
- **函数注册与发现**：后端支持注册任意函数，前端能够自动发现并获取函数元数据
- **函数调用执行**：前端能够调用后端函数，支持任意参数类型和返回值类型
- **本地通信机制**：实现前后端本地HTTP通信，支持跨域请求处理
- **流程执行管理**：支持单个函数执行和整个流程顺序执行

### 1.2 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Next.js 16.1.6, React 19.2.3, TypeScript |
| 图形库 | ReactFlow |
| UI组件 | shadcn/ui, Tailwind CSS |
| 后端框架 | Python 3.14, FastAPI, Uvicorn |
| 通信协议 | HTTP REST API |

## 二、后端服务实现

### 2.1 函数注册机制

使用Python装饰器实现函数注册功能，通过反射机制获取函数元数据：

```python
def register_function(func):
    """注册函数到系统"""
    import inspect
    sig = inspect.signature(func)
    # 获取参数信息...
    # 获取返回值类型...
    registered_functions[func.__name__] = {...}
    return func
```

### 2.2 API端点

- `GET /functions` - 获取所有注册函数
- `POST /execute` - 执行指定函数

### 2.3 示例函数

系统预置了三个示例函数供测试：
- `add(a: int, b: int) -> int` - 加法函数
- `multiply(a: int, b: int) -> int` - 乘法函数
- `greet(name: str) -> str` - 问候函数

## 三、前端应用实现

### 3.1 图形化节点编辑器

采用ReactFlow库实现拖拽式节点编辑器，支持：
- 节点拖拽移动
- 节点间连接
- 缩放和平移
- 流程图自动布局

### 3.2 节点类型设计

#### FunctionNode（函数节点）
- 蓝色边框设计
- 左侧输入连接点（蓝色）
- 右侧输出连接点（绿色）
- 显示函数名称和参数列表
- 执行后显示"已执行"标记

#### StartNode（起始节点）
- 紫色边框设计（紫色主题）
- 右侧输出连接点（紫色）
- 用于定义流程的初始参数
- 支持动态添加/删除参数

### 3.3 连接逻辑

- 使用smoothstep类型边
- 带动画效果
- 蓝色线条表示数据流向
- 从输出连接点拖拽至输入连接点完成连接

### 3.4 执行流程

1. 拓扑排序确定执行顺序
2. 从StartNode开始依次执行
3. 前一个节点的输出作为后一个节点的输入
4. 实时更新节点执行状态
5. 在终端面板显示执行日志

## 四、用户界面设计

### 4.1 布局结构

```
+------------------+------------------------------------------+------------------+
|                  |                                          |                  |
|   左侧菜单栏     |           中间流程图区域                 |   右侧配置面板   |
|                  |                                          |                  |
|  - 系统标题      |  +--------+       +--------+            |  - 节点名称     |
|  - 保存流程      |  | Start  |------>|  add   |            |  - 参数配置     |
|  - 已保存流程    |  +--------+       +--------+            |  - 执行结果     |
|  - 操作提示      |                                          |  - 错误信息     |
|                  |                                          |                  |
+------------------+------------------------------------------+------------------+
|                      底部终端（可折叠）                       |
+------------------------------------------------------------------+
```

### 4.2 交互设计

- **右键菜单**：在画布空白处右键点击，弹出添加节点菜单
- **节点选择**：拖拽节点时自动选中，显示在右侧配置面板
- **参数配置**：
  - 无连接：可直接在输入框填写参数
  - 有连接：输入框禁用，显示"参数将从连接节点获取"
- **执行控制**：
  - 单节点执行：点击"执行函数"按钮
  - 流程执行：点击"执行流程"按钮

### 4.3 终端面板

- 采用终端风格设计（深色背景）
- 包含窗口控制按钮样式
- 可折叠/展开
- 显示执行日志，带有命令提示符样式

- 从localStorage读取已保存流程
- 完整恢复节点和连接状态

## 六、使用指南

### 6.1 创建流程步骤

1. 右键点击画布，选择"Start (起始节点)"
2. 右键点击画布，选择所需函数节点
3. 从Start节点右侧拖拽连线到函数节点左侧
4. 在右侧配置面板设置Start节点的初始参数
5. 点击"执行流程"运行整个流程

### 6.2 添加新函数

在后端 `backend/app.py` 中使用装饰器注册：

```python
@register_function
def my_function(param1: str, param2: int) -> dict:
    """自定义函数"""
    return {"result": f"{param1} - {param2}"}
```

重启后端服务后，前端自动发现新函数。
